2. CONTENT UPLOAD (TEACHER ONLY)
==============================================

### 2.1 Upload Content
POST http://localhost:3000/api/content/upload
Authorization: Bearer {teacher_token}
Content-Type: multipart/form-data

Form Fields:
- title: "Algebra Basics" (string, required)
- description: "Introduction to algebraic equations" (string, optional)
- subject_id: 1 (number, required) [1=maths, 2=science, 3=english]
- file: <binary-file> (jpg/png/gif, max 10MB, required)
- start_time: "2026-04-26T09:00:00Z" (ISO timestamp, required)
- end_time: "2026-04-26T18:00:00Z" (ISO timestamp, required)
- rotation_duration: 5 (number in minutes, optional, default 5)

Example cURL:
curl -X POST http://localhost:3000/api/content/upload \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "title=Algebra Basics" \
  -F "description=Introduction to algebraic equations" \
  -F "subject_id=1" \
  -F "start_time=2026-04-26T09:00:00Z" \
  -F "end_time=2026-04-26T18:00:00Z" \
  -F "rotation_duration=5" \
  -F "file=@/path/to/image.jpg"

Expected Response (201 Created):
{
  "status": "success",
  "data": {
    "content": {
      "id": 1,
      "title": "Algebra Basics",
      "subject_id": 1,
      "status": "pending",
      "created_at": "2026-04-26T10:00:00.000Z"
    }
  }
}

⚠️  SAVE CONTENT ID → Needed for approval workflow

==============================================
TESTING ROTATION LOGIC
==============================================

Setup:
1. Create 3 content items for Maths (subject_id = 1)
   - Content A: 5 min duration, start: 09:00, end: 18:00
   - Content B: 3 min duration, start: 09:00, end: 18:00
   - Content C: 2 min duration, start: 09:00, end: 18:00

2. Approve all 3 contents

3. Current time: 09:17
   - Total cycle: 10 minutes
   - Position: 7 minutes into cycle
   - Active: Content B (should be shown)

4. Test:
   GET /api/schedule/live/{teacher_id}
   
   Should return Content B

5. Wait 3 minutes (time: 09:20)
   - Position: 0 minutes into cycle
   - Active: Content A (cycle restarted)

==============================================
POSTMAN WORKFLOW
==============================================

1. Import Postman-Collection.json into Postman

2. Set environment variables:
   - base_url: http://localhost:3000
   - teacher_token: (will be set by login)
   - principal_token: (will be set by login)

3. Run requests in order:
   ✓ Register Teacher
   ✓ Register Principal
   ✓ Login Teacher (saves token)
   ✓ Login Principal (saves token)
   ✓ Upload Content
   ✓ Get Pending Contents
   ✓ Approve Content
   ✓ Get Live Content
   ✓ Get My Schedule
